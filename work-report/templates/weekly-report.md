# 周报模板

为了更好地支撑月报、季报和年终总结的生成，建议按照以下格式编写周报。

## 推荐格式

```markdown
XXX 好，
    本周（YYYY.MM.DD - YYYY.MM.DD）

**本周重点工作**

1. **[项目/模块名称]**
   - 工作内容：具体做了什么
   - 进度：完成情况（如：100%、进行中 80%）
   - 成果/数据：产出物、量化数据（如：完成 X 个接口、处理 X 条数据）
   - 问题/风险：遇到的问题或潜在风险（可选）
   - 协作：涉及的协作方 @姓名（可选）

2. **[项目/模块名称]**
   - 工作内容：...
   - 进度：...
   - 成果/数据：...

**下周重点工作**

- [ ] [项目]：计划做的事情
- [ ] [项目]：计划做的事情
```

## 字段说明

| 字段 | 是否必填 | 说明 | 对报告生成的作用 |
| ---- | -------- | ---- | ---------------- |
| 时间范围 | 必填 | 本周的日期范围 | 筛选指定时间段的周报 |
| 项目/模块名称 | 必填 | 工作所属的项目或模块 | 按项目汇总工作 |
| 工作内容 | 必填 | 具体做了什么 | 描述工作详情 |
| 进度 | 推荐 | 完成百分比或状态 | 月报/季报的进度跟踪 |
| 成果/数据 | 推荐 | 量化的产出和效果 | 年终总结"用数据说话" |
| 问题/风险 | 可选 | 遇到的问题或风险 | 体现问题解决能力 |
| 协作 | 可选 | 涉及的协作方 | 体现跨团队协作 |
| 下周计划 | 推荐 | 下周重点工作 | 计划跟踪和连续性 |

## 示例

```markdown
马哥好，
    本周（2025.12.29 - 2025.12.31，请假半天）

**本周重点工作**

1. **AISE 项目**
   - 工作内容：与 @姜平 沟通项目文档结构和规范
   - 进度：文档框架已确定（100%）
   - 成果/数据：产出《AISE 文档规范 v1.0》，包含 5 大模块、20+ 文档模板
   - 协作：@姜平

2. **接口测试平台（Apifox）**
   - 工作内容：完成正式版部署，推广至各团队
   - 进度：部署完成（100%），推广中（30%）
   - 成果/数据：已部署到生产环境，覆盖后端、前端、测试 3 个团队
   - 协作：前端 @程浩 通知；测试 @程学文 通知

3. **代码影响范围追踪**
   - 工作内容：完成 MCP 协议支持，前端集成调试
   - 进度：MCP 协议（100%），前端集成（80%），Agent 调试中
   - 成果/数据：支持 Java、Python、Go 3 种语言的代码分析
   - 问题/风险：Agent 调试遇到上下文长度限制问题，需优化

**下周重点工作**

- [ ] AISE：规范文档整理并 MD 化
- [ ] AISE：跟进超级出海季需求
- [ ] 接口测试平台：完成《基于 Apifox 平台的接口协作规范》
- [ ] 代码影响范围追踪：跟进前端集成进度，解决 Agent 上下文问题
- [ ] 跟进 IDEA 插件 mic-dev-toolbox 审核状态
```

## 周报内容对各类报告的支撑

| 周报字段 | 月报 | 季报 | 年终总结 |
| -------- | ---- | ---- | -------- |
| 工作内容 | 汇总主要工作 | 汇总主要工作 | 业绩达成 |
| 进度 | 体现月度进展 | 体现季度进展 | 辅助 |
| 成果/数据 | 月度成果 | 季度成果 | 用数据说话 |
| 问题/风险 | 问题跟踪 | 问题解决 | 个人成长 |
| 协作 | 辅助 | 辅助 | 跨团队协作 |
| 下周计划 | 计划跟踪 | 计划跟踪 | 辅助 |
